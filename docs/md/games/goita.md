## ゲーム概要

石川県能登地方発祥の伝統的なボードゲーム。2対2のチーム戦で行う。

## ゲームルール

- 2対2のチーム戦（プレイヤー1 + AI vs AI + AI、または将来的に通信対戦）
- 駒は40枚: 王将×2・飛車×2・角行×2・金将×4・銀将×4・桂馬×4・香車×4・歩兵×8・し×2・ど×2・ね×2・ひ×2・こ×2・さ×2
- 各プレイヤーに10枚ずつ配布
- 先手プレイヤーが駒を1枚伏せて出す → 次のプレイヤーが同じ種類の駒で取る → 取ったプレイヤーが次の駒を出す
- 取れない場合はパス
- 自分のチームのパートナーが出した駒も取れる
- 手札を全て出し切ったチームが勝利

## 盤面仕様

- 中央に場（出された駒を表示）
- 4プレイヤーの配置（上下左右）
- 手札エリア（自分の手札は表向き、他のプレイヤーは裏向き）
- チームの表示（自分とパートナーを色で区別）

## ユーザー操作

- 手札の駒をタップ → 選択
- 「出す」ボタンタップ → 場に出す
- 取れる場合: 場の駒と同種の手札をタップ → 自動的に取る
- AIターンは自動進行

## ライフ・ゲームオーバー

- ライフ制なし（ゲーム性として不要）
- 勝敗はチーム単位

## オフライン対応

- **AIとの対戦はオフライン完結**
- 対人通信対戦はオンライン必須（将来実装）
- オフライン時は対人モードのボタンをグレーアウト表示

## AI実装方針

- 基本AI: 貪欲法（取れる駒があれば取る）
- 中級AI: ミニマックス法（2〜3手先読み）
- 強AI: MCTSまたはルールベース強化

## 難度定義

| 難度 | 対戦形式 | AI難度 |
|---|---|---|
| Easy | 1 vs 3（AIチーム弱） | 貪欲法AI |
| Normal | 2対2（AI2人） | ミニマックス2手 |
| Hard | 2対2（AI2人） | ミニマックス3手 |

## 型定義

```typescript
export type PieceType =
  | 'king' | 'rook' | 'bishop' | 'gold' | 'silver'
  | 'knight' | 'lance' | 'pawn' | 'shi' | 'do'
  | 'ne' | 'hi' | 'ko' | 'sa'

export interface Piece {
  type: PieceType
  faceUp: boolean
}

export interface GoitaState {
  hands: Piece[][]     // 4プレイヤーの手札
  board: Piece | null  // 場に出ている駒
  currentPlayer: number
  teams: [number, number][]  // チーム分け
}
```

## 実装上の注意点

- 駒の表示には日本語文字または専用アイコンを使用
- 能登の伝統ゲームという文化的背景を大切にしたUIデザイン
- 将来の通信対戦を見据え、GoitaState はシリアライズ可能な純粋オブジェクトとして設計する
